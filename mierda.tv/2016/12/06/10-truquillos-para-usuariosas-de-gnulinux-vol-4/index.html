10 “truquillos” para usuarios/as de GNU/Linux – Vol 4.

6 diciembre, 2016 En # SYSTEM

1. Search and replace desde la línea de comandos

 

En esta web se puede descargar el search and replace. Ideal para parsear bases de datos wordpress para por ejemplo cambiar urls a fuego en la base de datos que estén en http y ponerlas a https o por ejemplo para cambiar el dominio en toda la base de datos. Busca y reemplaza.

Algunas veces esto falla si son muchos los registros a modificar. Falla desde web pero se puede usar desde la consola.

Ejemplo:

php srdb.cli.php -h 192.168.8.224 -n nombrebasededatos -u usuario -p password -s 'dominioanterior.com' -r 'dominionuevo.com'

2. Ver las conexiones activas con ss

 

ss
ss -4t 
ss -4a
ss -p4to

 

3. Bang con keyword

 

Si hemos escrito ss -p4to y luego más comandos es posible ejecutar el mismo comando rapidamente usando un bang más keyword así:

!ss

4. Lo mismo pero sin ejecutarse usando control + R

 

Con control + r buscaremos en el historial de comandos. Cuando pulsamos control + r tendremos que introducir lo que queremos buscar, en este caso ss . Se autoconpletará pero no se ejecutará si no pulsamos intro/enter.

 

5. Apropos para encontrar un programa

 

Algunas veces no recordamos el nombre de algún programa pero si sabemos que por ejemplo tiene que ver con pdf

apropos pdf

Eso buscará en los nombres y descripciones de las paginas de los manuales.

 

6. Búsqueda pasiva de puertos abiertos usando Google

 

Buscando por ejemplo lo siguiente en un navegador:

site:”elbinario.net”:8080
site:”elbinario.net”:8000

Obtendremos resultados que indican que están abiertos o han estado abiertos. Esto se presta a bash scripting.

 

7.  Eliminar o crear archivos que empiezan con –

 

Creamos un archivo llamado “-archivo.txt” con touch para ilustrar el ejemplo en el directorio actual:

touch "./-archivo.txt"

El nombre del archivo es “-archivo” (sin las comillas) y eso supone que comienza con un – que es lo que usamos para introducir argumentos a los programas en la línea de comandos (normalmente).

Si quisiéramos borrarlo podríamos usar — para indicar que no es un argumento aunque comience por –

rm -- -archivo.txt

Crearlo de nuevo sin tener que indicar directorio actual con ./

touch -- -archivo.txt

Esto nos servirá para muchos programas siempre que trabajemos con archivos que comienzan por -. Lo normal es intentar evitar esos nombres para los archivos.

 

8. Crear un directorio y subdirectorios dentro del tirón

 

Supongamos que quieres crear un directorio llamado 15905 y dentro de ese directorio otros 2 llamados backups y paquetes. Se puede hacer así:

mkdir -p 15905/{backups,paquetes}

El uso de -p y los braces lo hace posible.

 

9. free y vmstat para ver el consumo de memoria rápidamente

vmstat
free

 

10. Información sobre un paquete en debian (mantenedor, descripción, …)

apt-cache show htop
apt-cache show nmon
apt-cache show nmap
