Rdesktop – Control remoto de máquinas windows

5 agosto, 2016 En # CODE

Algunas veces algunas empresas suelen tener servidores con windows. Suelen ser proyectos viejos que en su momento algún inútil puso a funcionar sobre una maquina windows y la empresa como estaban ganando dinero y todo eran vacas gordas compraron licencias pensando que estaban invirtiendo en algo bueno ya que eran caras.

Compraron licencias para el sistema y luego licencias para determinado software muy especifico. Eso llevó a la empresa a depender del software propietario en determinadas cosas. Todos los años pagan religiosamente su licencia para que aquello no deje de funcionar. El soporte ya no se da para la versión que compraron hace años y si quieren soporte van a tener que migrar a la nueva versión.

Esa migración no es sencilla ya que la nueva versión es totalmente incompatible con la que ellos compraron.

Al final ocurre que se necesita seguir manteniendo esas máquinas y aunque es odioso salir de la línea de comandos y administrar un jodido windows así no queda otra. No queda otra ya que los aplicativos que adquirieron son de “clic aquí, clic allá” y no están pensados para que administradores de sistemas trabajen automatizando tareas (no tienen posibilidad de trabajar vía cli).

 

 

Estos motivos nos llevan algunas  veces a buscar software libre que nos permita poder trabajar con esas viejas máquinas con ese software obsoleto (muchas de las empresas de algunos programas ya no existen) y poder administrarlo mientras el cuerpo aguante.

Rdesktop vino a salvarnos

 

rdesktop es crema y es la herramienta que entra en juego para estas tareas. Su web es sencilla: http://www.rdesktop.org/

Ocupa poco y consume pocos recursos. Ideal para crearnos alias con cada una de las maquinas y tener acceso rápidamente a las maquinas a las que deseamos entrar. Un ejemplo de alias que podemos meter en el archivo .bashrc son estos:

alias remoto-qlickpro='rdesktop  -u administrador -p "lapassword" -k es -g 1224x900 -a 8 -z -x l -0 -5 -r sound:off -T "Qlick maquina" 192.168.222.111'
alias remoto-proyectonoseque='rdesktop  -u administrador -p "lapassword" -k es -g 1224x900 -a 16 -z -x l -0 -5 -r sound:off -T "Proyecto uno, el tiburón" 192.168.222.112'
alias remoto-virtualcentervmware='rdesktop  -u administrador -p "lapassword" -k es -g 1224x900 -a 16 -z -x l -0 -5 -r sound:off -T "VIRTUALCENTER" 192.168.222.113'

Luego se actualiza el .bashrc así:

source /home/tuusuario/.bashrc

Y ahora si escribimos remoto y pulsamos tabulador tendremos a mano las 3 maquinas que hemos añadido.

 

 

He probado diferente software para esto y este es sinceramente el que más me ha convencido. Se abre rápidamente y se pueden limitar colores y especificar comprensión para momentos en los que tengamos conexión lenta.

 

Remote Desktop Protocol (RDP) es un protocolo propietario desarrollado por Microsoft que permite la comunicación en la ejecución de una aplicación entre un terminal (mostrando la información procesada que recibe del servidor) y un servidor Windows (recibiendo la información dada por el usuario en el terminal mediante el ratón ó el teclado).

 

Para poner a pantalla completa o salir de la pantalla completa en rdesktop podemos usar la combinación de teclas Ctrl + Alt + Enter .

Si necesitamos un gestor gráfico en vez de tenerlo en el .bashrc podemos instalar gnome-rdp (está en los repositorios de debian por ejemplo).

No deja de ser un programa para ir guardando conexiones a máquinas y tenerlo luego a golpe de clic. Nada que no se pueda hacer con un alias mejor. Gnome-rdp tira finalmente de rdesktop.
