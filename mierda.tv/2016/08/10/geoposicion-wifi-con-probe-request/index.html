Geoposición wifi con probe request

10 agosto, 2016 En # CODE

Poner en modo monitor una antenita wireless en GNU/Linux es sencillo. Por ejemplo así:

# ifconfig ra0 up
# iwconfig ra0 mode monitor

 

Siendo ra0 el nombre de la interface. Por ejemplo la tuya podría ser eth0 . Para realizar la tarea necesitaremos instalar la suite aricrack y tcpdump. En debian puede hacerse así:

 

# apt-get install aircrack-ng

 

Nos ponemos a escanear un poco almacenando la chicha en un .cap así:

 

# airodump-ng ra0 -w mierda

 

Con el tcpdump en otra terminal así:

 

# tcpdump -n -l -e -r mierda-01.cap | grep 'Probe Request ([^)]' | cut -d " " -f 2-15 | cut -d " " -f 1,6 | sed 's/(//' | sed 's/)//' | sort | uniq

 

Estaremos viendo un montón de sitios a los que la gente se ha conectado. por ejemplo estos:

Esto es un rollo. Vamos a lo bueno

 

Lo bueno es que existe wigle y existe mucha gente mandando información que puede ser consultada mediante una API.

Con wigle en python podemos consultar. Instalar esto es sencillo:

 

# apt-get install python-pip
# pip install --upgrade pip
# pip install wigle

 

Si quieres también puedes clonar el repo y buscarte la vida para instalarlo:

$ git clone https://github.com/viraptor/wigle
$ cd wigle
# apt-get install python3-setuptools
# python3 setup.py build
# python3 setup.py install

 

Ahora a usarlo para algo productivo. Por ejemplo para esto:

 

wigle_search --user nombreusuario --pass lapassword --bssid "9c:1c:12:89:78:a4"

 

No olvides que has de tener cuenta en wigle para poder atacar la API o usar la aplicación que puedes bajar de F-droid.
