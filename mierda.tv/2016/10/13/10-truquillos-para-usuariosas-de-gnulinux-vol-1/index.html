10 “truquillos” para usuarios/as de GNU/Linux – Vol 1.

13 octubre, 2016 En # SYSTEM

La experiencia que va ofreciendo el día a día tiene muchas veces como consecuencia el aprender nuevas formas de hacer algo o simplemente más efectivas. Otras veces el día a día nos va nutriendo de pequeños “trucos” o formas de hacer que nos hacen la vida más sencilla.

Con el tiempo he ido recopilando algunas que iré soltando en diferentes posts. Son estas (seguramente muchas de ellas tu ya las conocías pero lo mismo a alguien le resulta alguna de utilidad):

 

  1. Usar “less + f ” en vez de “tail -f” en los archivos de logs.

  2. Comprimir tar.gz y descomprimir:

    tar xfvz --> extraer
tar cfvz --> comprimir

  3. Lanzar una query mysql larga y obtener la salida en un CSV

    mysql -u user -ppassword basedatis < archivoconlaquery.sql > salida.csv

  4. Contraseña por defecto de un DSET de Dell = dell

  5. Excluir contenido de un directorio al realizar un tar.gz

    tar cfvz /home/usuario/archivo.tar.gz --exclude "/ruta/*" .

  6. Generar pruebas de stress para emular load average

    Con cat: cat /dev/zero > /dev/null
    Con stress: apt-get install stress ; stress --cpu 2 --io 1 --vm 1 --vm-bytes 512MB --timeout 40s

  7. Partir en volumenes ARJ para comprimir

    apt-get install arj; arj a -r -y -v950M gonzalo.arj gonzalo/

  8. Sincronizar hora con NTP server

    ntpdate -u servidortime

  9. Mostrar porcentaje al importar base de datos

    apt-get install pv

pv bk-thinkbig.sql | mysql -u root -ppassword

    Otros ejemplos:

    dd if=/lugar/de/origen | pv | dd of=/lugar/dedestino/

  10. Dump de base de datos en producción

    mysqldump -u root --single-transaction --quick nombrebasededatos | gzip > bk-basededatos.sql.gz
