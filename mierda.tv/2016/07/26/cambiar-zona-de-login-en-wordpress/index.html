Cambiar zona de login en wordpress

26 julio, 2016 En # SYSTEM

No es mala cosa cambiar la zona en la que hacemos login en wordpress por otra diferente, es decir, wp-admin y wp-login suelen ser las zonas por las que hacer login a un wordpress por defecto. Es buena cosa cambiar esto para evitar intentos de acceso automatizados.

Creamos un directorio llamado nuevazonaadmin en la raíz del directorio donde tengamos desplegado el wordpress.

En mi caso lo he realizado en un servidor con nginx. Si usas apache la cosa cambia pero es más o menos lo mismo.

mkdir nuevazonaadmin
chown www-data www-data nuevazonaadmin

Si no queremos poner nuevazonaadmin pondremos el nombre del directorio secreto que queramos. Ese nombre lo fijamos en el archivo wp-config.php con estas 2 lineas:

define('WP_ADMIN_DIR', 'nuevazonaadmin');
define( 'ADMIN_COOKIE_PATH', SITECOOKIEPATH . WP_ADMIN_DIR);

Ahora hemos de añadir lo siguiente al archivo functions.php que esté en el theme que tengamos activo. Este estará normalmente en el directorio /wp-content/themes/nombretheme/

add_filter('site_url',  'wpadmin_filter', 10, 3);

function wpadmin_filter( $url, $path, $orig_scheme ) {
        $old  = array( "/(wp-admin)/");
        $admin_dir = WP_ADMIN_DIR;
        $new  = array($admin_dir);
        return preg_replace( $old, $new, $url, 1);
}
add_action('login_form','redirect_wp_admin');

function redirect_wp_admin(){
        $redirect_to = $_SERVER['REQUEST_URI'];
        if(count($_REQUEST)> 0 && array_key_exists('redirect_to', $_REQUEST)){
                $redirect_to = $_REQUEST['redirect_to'];
                $check_wp_admin = stristr($redirect_to, 'wp-admin');
                if($check_wp_admin){
                        wp_safe_redirect( '404.php' );
                }
        }
}

 

Finalmente toca configurar el virtualhost en el archivo de configuración de nginx:

location ~* /nuevazonaadmin/ {

rewrite ^/nuevazonaadmin/(.*) /wp-admin/$1 last;

}
