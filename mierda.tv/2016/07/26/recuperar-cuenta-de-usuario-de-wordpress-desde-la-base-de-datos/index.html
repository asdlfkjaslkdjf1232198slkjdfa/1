Recuperar cuenta de usuario de wordpress desde la base de datos

26 julio, 2016 En # SYSTEM

Se puede dar el caso de que no recordemos la contraseña de un usuario de wordpress y que al mismo tiempo tengamos deshabilitada la función mail() de php en el servidor.

Esto no nos permitirá recuperar la contraseña con el método tradicional de recuperación de password cuando la hemos olvidado y por tanto tendremos que buscar otras formas.

La forma que a mi me funciona es la de cambiar el hash de la password por otro en la base de datos. No recuperamos la password pero si que podremos colocar otra para poder acceder con ese usuario.

La forma de hacer esto es así:

  • Entramos en el servidor vía ssh (por ejemplo) y miramos el usuario, password y nombre de base de datos que está usando wordpress ya que estará en el archivo wp-config.php
  • Accedemos a la base de datos así: mysql -u usuario -pPASSWORD nombrebasededatos
  • Miramos cuantos usuarios existen en la instancia de wordpress así:
    SELECT ID, user_login, user_pass FROM wp_users;
  • Generamos un hash md5 para una nueva password así:
    echo "nuevapassword" | tr -d '\r\n' | md5sum | cut -d " " -f 1
  • Se puede generar uno también desde aquí: http://www.miraclesalad.com/webtools/md5.php
  • Nos copiamos el viejo hash que hemos visualizado con la sentencia SELECT a algún lado para no perderlo ya que quizas nos interese dejar la cuenta como estaba.
  • Actualizamos el nuevo hash incluso cuando el viejo esta salteado. WordPress nos permitirá hacer login sin problema. Esto lo hacemos así:
    UPDATE wp_users SET user_pass="4d186321c1a7f0f354b297e8914ab240" WHERE ID=1;

Hemos de cambiar el hash por el generado por nosotros. En mi caso he puesto “hola” de password y me ha salido ese hash. Hemos de cambiar el ID por el que queramos cambiar, es decir, si el ID del usuario es 6 pues tendremos que especificarlo :).
