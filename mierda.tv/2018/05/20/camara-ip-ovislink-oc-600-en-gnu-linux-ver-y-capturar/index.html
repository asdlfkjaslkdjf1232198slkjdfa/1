Cámara IP Ovislink OC-600 en GNU/Linux – Ver y capturar

20 mayo, 2018 En # SYSTEM

Se trata de un modelo de Cámara IP de hace ya unos años y puede servir para video vigilancia perfectamente. Tiene capacidad de mandar capturas a un email cuando detecta movimiento por ejemplo o subirlas vía FTP a algún server.

Se puede configurar/gestionar desde el navegador. La IP por defecto en caso de reseteo de fabrica es la 192.168.0.99 .
Para resetearlo se puede dejando presionado el botón trasero con un bolígrafo o un clip doblado.

Para acceder al stream de la cámara es posible con ffplay así:

$ ffplay rtsp://192.168.1.4:554/live.sdp

Desde la web se puede configurar el nombre y en vez de live.sdp que sea otra cosa. La IP puede cambiarse también por otra. Esa es solo un ejemplo. El puerto es 554 y el protocolo RTSP.

Para grabar se puede hacer así:

$ ffmpeg -i rtsp://@192.168.1.4:554/live.sdp -acodec copy -vcodec copy /home/fanta/salida.mp4

Hoy me he traído 2 de estas cámaras a casa para ponerlas en orden y con suerte podrán servir en un futuro próximo para poder realizar grabaciones de objetos 3D desde diferentes ángulos por ejemplo. Ya que estamos a la espera de que llegue una impersora 3D al hacklab 301 del centro social La Ingobernable.

Nota1: Es posible que la cámara este configurada para obtener una IP por DHCP. En ese caso el server DHCP será el que le reparta una IP.
Nota2: Si no nos interesa capturar audio podemos mutear eso desde la configuración usando el navegador.
Nota3: Es posible indicar que el led esté apagado. Eso es de utilidad para no meter luz de colorines auzl y roja sin venir a cuento en lo que vamos a grabar. En caso de que sea una cámara de vigilancia puede ser de utilidad que no lleve el LED. NO para espiar a gente sin su consentimiento pero si de algún modo para evitar molestias.
