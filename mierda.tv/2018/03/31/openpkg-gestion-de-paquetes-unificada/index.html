Openpkg – Gestión de paquetes unificada

31 marzo, 2018 En # SYSTEM

OpenPKG es un gestor de paquetes para Unix basado en el sistema de paquetería RPM. La idea es que unifica la instalación de paquetes en diferentes tipos de plataformas Unix (Solaris, Linux, FreeBSD).

La web del proyecto: http://www.openpkg.org/.

Para instalarlo:

$ wget  http://openpkg.org/go/download/openpkg.src.sh
# sh openpkg.src.sh --prefix=/openpkg --tag=openpkg --user=openpkg --group=openpkg

Tardará un buen rato y terminará con este mensaje si todo va bien:

Tras esto meteremos el comando que nos salga. En mi caso este:

# sh openpkg-4.8.19-20180101.amd64-debian9.3-openpkg.sh

Y si queremos podemos añadir al $PATH “/openpkg/bin” para disponer de openpkg

PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/openpkg/bin

Para por ejemplo instalar una versión más moderna de bash se puede hacer así:

# /openpkg/bin/openpkg build bash | sh

Para buscar por ejemplo gcc así:

# /openpkg/bin/openpkg search gcc

Para compilar gcc 7.2 así:

# /openpkg/bin/openpkg build gcc | sh
