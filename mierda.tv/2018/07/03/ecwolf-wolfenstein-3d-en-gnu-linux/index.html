ecwolf – Wolfenstein 3D en GNU/Linux

3 julio, 2018 En # GAMES

En 1999 ID software liberó el código del engine de Wolfestein 3D. Desde entonces han surgido un montón de forks siendo ecwolf un fork de otro fork llamado wolf4sdl y siendo uno de los más actuales e interesantes.

La web del proyecto es: http://maniacsvault.net/ecwolf/

Lo interesante es que compilando el tinglado solo necesitaremos un solo binario para ejecutar el juego (y los datos del juego).
Vamos a poder jugar al juego con WASD + ratón y con un montón de resoluciones posibles.

La forma de compilar y tener todo listo para jugar es esta (la que yo he usado):

# apt-get install libsdl1.2-dev libsdl-image1.2-dev libsdl-mixer1.2-dev libsdl-ttf2.0-dev cmake libgtk2.0-dev

$ wget "http://maniacsvault.net/ecwolf/files/ecwolf/1.x/ecwolf-1.3.3-src.tar.xz"
$ tar xfvJ ecwolf-1.3.3-src.tar.xz 
$ cd ecwolf-1.3.3-src/
$ mkdir build
$ cd build
$ cmake ..
$ make
$ mkdir juego
$ cd juego
$ cp -pRv ../ecwolf* .
$ wget "https://mierda.tv/wp-contenido/uploads/2018/06/wolfesnstein3D_data.tar.gz"
$ tar xfvz wolfesnstein3D_data.tar.gz
$ ./ecwolf

Lo que viene a ser instalar dependencias (en debian 9 con apt, en otros sistemas como corresponda) y luego compilar. Finalmente se ve que se baja un archivo con los datos del juego original(una de tantas versiones, pero la que cuadra con este motor).

Este juego sinceramente al día de hoy jugarlo es un poco rollo. No obstante supuso la base de toda una revolución con su motor de 2.5D que en realidad era 2D falseado.
Estiraban las paredes pero era un falso 3D.

Este juego aún no tiene texturas para suelo y cielo. No obstante es el inicio y conocer la historia de como lo desarrollaron es fascinante :).
