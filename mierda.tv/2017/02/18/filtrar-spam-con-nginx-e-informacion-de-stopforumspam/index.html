Filtrar spam con Nginx e información de stopforumspam

18 febrero, 2017 En # SYSTEM

Combatir el spam en wikis, wordpress, … gracias a stopforumspam es sencillo y se puede hacer directamente desde la configuración del servidor web. Esto facilita mucho la vida y limita mucho la de los spamers. Seguirás teniendo spam de vez en cuando pero este será muchísimo menos.

 

mkdir -p /root/scripts
cd /root/scripts/
wget https://cdn.content-network.net/nbl/nbl.sh
chmod +x nbl.sh
crontab -e

Guardamos lo siguiente cuando se abra el editor de textos que tengamos definido:

 

0 1 * * * cd /root/scripts && ./nbl.sh >> /var/log/nbl.log

A la 1 todos los días se ejecutará el script para bajarse nuevas ips que denegar.
Si miramos el script nbl.sh veremos que almacenará las reglas en /etc/nginx/conf.d/  y podemos comprobar como funciona ejecutándolo antes de meterlo en cron.
