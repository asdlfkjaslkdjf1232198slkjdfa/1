Montar gdrive como un directorio y ofrecer acceso mediante Apache

8 abril, 2017 En # SYSTEM

Yo no tengo cuenta de google pero entiendo que muchas veces en algunos trabajos obligan a la gente a usar google drive. Las propias empresas te dan una cuenta de google y te dicen que uses eso.

La cosa es que podemos montar el tinglado en un VPS y utilizar ese espacio al menos temporalmente para compartir cosejas o para un backup cifrado de algo.

La cosa es que para almacenar información que no sea privada y cosas así puede servir esta receta:

# apt-get install ocaml ocaml-findlib sqlite3 camlidl m4 libcurl4-gnutls-dev libfuse-dev libsqlite3-dev camlp4-extra fuse build-essential sudo ssh make git libsqlite3-ocaml-dev libsqlite3-ocaml opam
$ opam init
$ eval `opam config env`
$ opam install google-drive-ocamlfuse

En tu home tendrás el software listo para usarse en “.opam/system/bin/”

La primera vez que lo ejecutas necesitas un navegador. Lo que se puede hacer es instalarlo en local también y usar el navegador para validar contra google drive. Luego copias el directorio .gdfuse de tu home al servidor y montas así por ejemplo:

./google-drive-ocamlfuse -o allow_other,default_permissions,umask=022 /var/www/html/almacen/
